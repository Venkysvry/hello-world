Angular is a popular front-end framework developed by Google, primarily used for building single-page applications (SPAs). It provides a robust structure for developing dynamic web apps with features like two-way data binding, dependency injection, and modular architecture.

The first version of Angular (AngularJS) was released in 2010 and used JavaScript as its programming language.
Later versions, starting from Angular 2, transitioned to TypeScript, a statically typed superset of JavaScript that offers better tooling, type safety, and object-oriented features.
Role of Node.js in Angular Development:

While Angular itself runs in the browser, Node.js plays a crucial role in the development environment:

Development Tools: Angular CLI (Command Line Interface), which is used to create, build, and manage Angular projects, is built on Node.js.
Package Management: Node.js includes npm (Node Package Manager), which is used to install Angular libraries, third-party modules, and development dependencies.
Build Process: Node.js enables tools like Webpack, TypeScript compiler, and Babel to bundle and transpile Angular code efficiently.
Server-Side Rendering (SSR): With Angular Universal, Node.js can be used to render Angular applications on the server side, improving performance and SEO.
Local Development Server: Node.js powers the local development server that serves your Angular app during development, enabling live reload and fast feedback.

DOM = Document Object Model

The DOM is a tree‑like JavaScript representation of an HTML page that the browser creates and manages.

In short:

HTML → parsed by browser

JavaScript (and Angular) interacts with the DOM, NOT raw HTML

In Angular, decorators are special TypeScript functions that add metadata to classes, properties, methods, or parameters. Angular uses this metadata to understand how different parts of your application should behave at runtime.

Class decorators are applied to classes and define what role the class plays in an Angular application.
Common Class Decorators

Creates a custom pipe.

export class CapitalizePipe {}

@Injectable
Marks a class as available for dependency injection.
@Injectable({
  providedIn: 'root'
})
export class UserService {}

2. Property Decorators
Property decorators are applied to class properties and control data flow between components and directives.
Common Property Decorators
@Input
Receives data from a parent component.
@Input() userName!: string;

@Output
Emits events to a parent component.
@Output() userSelected = new EventEmitter<number>();
@HostBinding
Binds a property to a host element attribute.
@HostBinding('class.active') isActive = true;

3. Method Decorators
Method decorators are applied to class methods and typically handle events.
Common Method Decorators
@HostListener
Listens to events on the host or global objects.
@HostListener('click')
onClick() {
  console.log('Element clicked');
}
✅ Used for: Event handling

4. Parameter Decorators
Parameter decorators are applied to constructor parameters and control dependency injection behavior.
Common Parameter Decorators
@Inject
Specifies a custom provider token.
constructor(@Inject(APP_CONFIG) config: AppConfig) {}

@Optional
Marks a dependency as optional.
constructor(@Optional() private logger?: LoggerService) {}
@Self
Forces Angular to resolve dependency from the local injector.
constructor(@Self() private service: LocalService) {}

Decorators are metadata annotations in Angular that tell the framework how to process classes, properties, methods, and constructor parameters.

Angular Change Detection is the mechanism that tracks changes in component state and updates the DOM accordingly.
By default, Angular uses Zone.js, which triggers change detection whenever asynchronous events occur.
Signals, introduced in Angular 16, provide a fine‑grained, reactive way to track state changes without relying on Zone.js. Signals are accessed using function syntax and updated using set() or update().
Signals work best with OnPush change detection and can be used in a zoneless Angular setup for improved performance.
OnPush improves performance by limiting change detection to specific conditions, while Default checks the whole component tree on every event.
 Real‑World Analogy

Default → Checking every room when one bulb breaks
OnPush + Signals → Checking only the broken bulb

What is the use of AuthGuard in Angular?
An AuthGuard in Angular is used to control access to application routes based on conditions such as whether a user is logged in, authorized, or has a specific role.
In simple terms:
👉 AuthGuard protects routes from unauthorized users

✅ Why do we need AuthGuard?
Without an AuthGuard:

Any user can directly access protected URLs
(example: /dashboard, /admin, /profile)

With an AuthGuard:

Only authenticated or authorized users can access specific routes
Others are redirected (usually to /login)


✅ How AuthGuard Works (Flow)

User tries to navigate to a route
Angular checks the AuthGuard
Guard returns:

✅ true → allow access
❌ false → block access
🔁 Redirect to login or another page




✅ Example: AuthGuard Logic
🔹 auth.guard.ts
import { Injectable } from '@angular/core';
import { CanActivate, Router } from '@angular/router';

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {

  constructor(private router: Router) {}

  canActivate(): boolean {
    const isLoggedIn = !!localStorage.getItem('token');

    if (isLoggedIn) {
      return true;
    } else {
      this.router.navigate(['/login']);
      return false;
    }
  }
}
``Checks whether user is logged in
✅ Redirects to login if not

✅ Using AuthGuard in Routes
{
  path: 'dashboard',
  component: DashboardComponent,
  canActivate: [AuthGuard]
}
✅ Protected route
✅ Cannot access directly without login

✅ Types of Guards in Angular
GuardPurposeCanActivateAllow or block route accessCanActivateChildProtect child routesCanDeactivatePrevent leaving a pageResolveFetch data before route loadsCanLoadPrevent lazy module loading

✅ Real‑World Example

/login → public
/dashboard → only logged‑in users
/admin → only admin users

AuthGuard ensures:

Security ✅
Proper navigation ✅
Better user experience ✅
